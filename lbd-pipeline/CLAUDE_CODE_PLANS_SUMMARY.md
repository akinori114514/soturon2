## 2. 方法論

### 2.1 Literature-Based Discovery（LBD）の採用理由

**LBDとは**: Don R. Swansonが1986年に提唱した、異なる文献群に分散した知識を統合することで新たな仮説を発見する手法。

**採用理由**:
1. **文献の分散性**: SCS研究は法学、政治学、情報科学、経営学など複数分野にまたがる
2. **日中比較の困難さ**: 両者を橋渡しする研究が少ない
3. **体系的整理の必要性**: 91件のPDFから概念を網羅的に抽出する作業は人手では非効率

### 2.2 SwansonのABCモデル

```
A ─────────── C
 \           /
  \    ?    /
   \       /
    \     /
      B

A: 既知の概念群（例: 中国SCS制度）
B: 中間概念（例: データフロー設計）
C: 別の既知概念群（例: 日本信用情報機関）
?: 未検出の関係（研究仮説候補）
```

**優先検出パターン**:
1. 中国SCS ⇔ 日本制度のギャップ
2. 理論的フレームワーク ⇔ 実証事例のギャップ
3. EU/US規制 ⇔ 中国SCSのギャップ

### 2.3 超伝導体論文（Itani et al. 2025）との対応

本パイプラインは、arXiv:2512.10847のEnd-to-End LLMパイプラインを社会科学研究に応用したもの：

| 超伝導論文の要素 | 本プロジェクトでの対応 |
|------------------|----------------------|
| 90,000 DOIs | 91件のPDF |
| composition, Tc, pressure | 概念、4軸分類、研究領域 |
| SC/NSC分類 | 概念タイプ分類 |
| Tc予測 | ギャップスコア予測 |
| 逆設計（Tc→composition） | 仮説生成（ギャップ→研究問い） |
| GPT-4o抽出 | Claude抽出 |

### 2.4 先行研究との位置づけ

| 先行研究 | 手法 | 本研究との関係 |
|---------|------|---------------|
| Swanson (1986) | 文献間の間接的関連発見 | ABCモデルをSCS研究に適用 |
| Itani et al. (2025) | LLMによるEnd-to-End抽出 | パイプライン設計の参照 |
| 福田ら (2024) | LLMによる知識グラフ自動構築 | Sentence-BERT重複検出の参照 |
| Bi (2021) | SCSの2層構造分析 | 概念タイプ定義の参照 |
| Liang et al. (2018) | SCSの3フェーズモデル | データフロー軸の設計参照 |

## 3. 実装詳細

### 3.1 パイプライン設計

```
[Phase 1] PDF → pdftotext → テキスト
[Phase 2] テキスト → 3000字チャンク分割
[Phase 3] チャンク → 概念抽出 + ラベル付与
[Phase 4] 概念DB統合 → 知識グラフ生成
[Phase 5] マルチタスク予測（Task A/B/C）
[Phase 6] ギャップ検出（Swanson ABC）+ 仮説生成
[Phase 7] 可視化 + RQマッピング
```

### 3.2 概念抽出スキーマ

```json
{
  "concept_id": "SCS-0001",
  "jp_name": "連合懲戒",
  "en_name": "Joint Punishment",
  "zh_name": "联合惩戒",
  "concept_type": "mechanism | theory | phenomenon | actor | policy | data_system",
  "definition": "定義",
  "four_axis": ["インセンティブ設計", "ガバナンス"],
  "source_papers": ["bi_2021", "liang_2018"],
  "context_quotes": ["原文引用"],
  "related_concepts": ["関連概念ID"]
}
```

### 3.3 概念タイプの定義（6種類）

| タイプ | 説明 | 例 |
|--------|------|-----|
| mechanism | 制度メカニズム | 連合懲戒、連合激励、信用修復 |
| theory | 理論的概念 | 4軸フレームワーク、逆適応 |
| phenomenon | 観察される現象 | 失信被執行人、ブラックリスト掲載 |
| actor | 行為者・組織 | NDRC、PBOC、アリババ |
| policy | 政策・法令 | 2014年綱要、征信業管理条例 |
| data_system | データシステム | NCISP、Credit China |

### 3.4 エッジタイプの定義（5種類）

| 関係タイプ | 説明 | 重み付け |
|-----------|------|---------|
| co_occurrence | 同一文献内共起 | 共起回数 |
| causal | 因果関係 | 明示的因果表現の有無 |
| comparison | 比較関係 | 比較対象として言及 |
| implementation | 実装関係 | 制度が他の制度を実装 |
| subsumption | 包含関係 | 上位概念-下位概念 |

### 3.5 抽出プロンプト

```
以下のテキストから社会信用システム関連の概念を抽出してください。

【抽出基準】
- SCS（社会信用システム）に直接関連する概念
- 日本・EU・USの比較対象となる制度概念
- 4軸（ガバナンス、データフロー、インセンティブ設計、官民関係）に分類可能な概念

【除外基準】
- 一般的な用語（「データ」「情報」「規制」等）
- 文脈なしでは意味をなさない概念
- 出典不明の主張

【出力形式】JSON
```

## 8. 結論

本プロジェクトでは、Claude Codeを使用して91件のPDF文献から627件の概念を抽出し、584件のエッジで構成される知識グラフを構築した。Swanson ABCモデルを適用し、50件のギャップ候補と20件の研究仮説を生成した。

**達成された成果**:
- 概念抽出: 目標500件に対し627件（125%達成）
- 処理率: 目標90%に対し97%達成
- ギャップ検出: 日中間の比較分析に有用な仮説を生成

**残された課題**:
- Task A/B/C予測結果のデータベースへの反映
- RQレポート生成スクリプトの修正
- 孤立概念の接続強化（目標100件以下に対し現状210件）

プロジェクトは概念抽出と知識グラフ構築の基盤を確立したが、予測タスクの結果反映とRQマッピングの完成が今後の課題である。
