# 付録：Literature-Based Discovery（LBD）分析手法

本付録では、本研究で用いた文献ベース発見（LBD）分析の手法を説明する。

---

## A. 手法概要

### A.1 Literature-Based Discoveryとは

Literature-Based Discovery（LBD）は、Swanson（1986）が提唱した知識発見手法である。異なる研究領域の文献間に存在する暗黙の関連性を、明示的に接続することで新たな仮説を導出する。

**Swanson ABCモデル**：
```
文献群A ─────────── 文献群C
    \                   /
     \       ?         /
      \               /
       \             /
         概念B（橋渡し）

A: 既知の概念群
B: 橋渡し概念（中間概念）
C: 別の既知概念群
?: 未検出の関係（研究仮説候補）
```

本研究では、91件のSCS関連文献から概念を抽出し、知識グラフを構築することで、4軸フレームワーク分析（RQ1）、企業影響分析（RQ2）、日本への示唆（RQ3）に資するエビデンスを体系化した。

### A.2 先行研究との対応

本手法は、Itani et al.（2025）による超伝導材料発見パイプラインを社会科学研究に応用したものである。

| Itani et al.（2025） | 本研究 |
|---------------------|--------|
| 90,000 DOIs → 構造化DB | 91 PDFs → 概念DB |
| SC/NSC分類 | 企業影響分類（便益/コスト/リスク） |
| Tc予測（回帰） | 4軸スコア回帰 |
| GNoMEスクリーニング | Swanson ABCギャップ検出 |

## B. パイプライン設計

### B.1 処理フロー

```
[Phase 1] PDF → テキスト抽出
    ↓ pdftotext による変換
[Phase 2] テキスト → チャンク分割
    ↓ 3,000字単位で分割（コンテキスト管理）
[Phase 3] チャンク → 概念抽出・ラベル付与
    ↓ LLMによる構造化抽出
[Phase 4] 概念DB統合 → 知識グラフ生成
    ↓ 共起・因果・比較関係の検出
[Phase 5] ギャップ検出 → 仮説生成
    ↓ Swanson ABCモデルの適用
[Phase 6] RQマッピング → 成果物生成
```

### B.2 コンテキスト管理

91件のPDF（総計約510万字）を直接処理するとコンテキスト爆発が生じるため、以下の設計原則を適用した。

| 原則 | 内容 |
|------|------|
| 即時永続化 | 抽出結果は即座にファイル保存し、コンテキストに蓄積しない |
| バッチ分割 | 大きな処理は固定サイズバッチに分割 |
| ステートレス再開 | progress.jsonで状態管理し、中断後も再開可能 |
| 1文献1セッション | 大規模文献（100チャンク超）は独立セッションで処理 |

### B.3 概念抽出スキーマ

各概念は以下のスキーマで構造化した。

```json
{
  "concept_id": "SCS-0001",
  "jp_name": "連合懲戒",
  "en_name": "Joint Punishment",
  "zh_name": "联合惩戒",
  "concept_type": "mechanism",
  "definition": "一箇所の信用失墜が複数領域に波及する制裁メカニズム",
  "four_axis": ["インセンティブ設計", "ガバナンス"],
  "source_papers": ["bi_2021", "liang_2018"],
  "country_tag": "CN"
}
```

## D. 概念分類定義

### D.1 概念タイプ（concept_type）

| タイプ | 定義 | 例 |
|--------|------|-----|
| theory | 理論・概念枠組み | 市場信用、公共信用、xingyong |
| mechanism | 制度メカニズム | 連合懲戒、信用修復、分類規制 |
| policy | 政策・法規 | 社会信用体系建設計画綱要（2014-2020） |
| data_system | データシステム | 信用中国、芝麻信用、統一社会信用コード |
| actor | アクター | NDRC、人民銀行、最高人民法院 |
| phenomenon | 現象・状況 | 三角債務、信用ギャップ |

### D.2 エッジタイプ（関係タイプ）

| タイプ | 定義 | 検出方法 |
|--------|------|----------|
| co_occurrence | 同一文献内での共起 | 同一チャンク内での出現 |
| reference | 定義文内での参照 | 定義文に他概念名が含まれる |
| subsumption | 包含関係（上位-下位） | is-a関係のヒューリスティック推定 |
| causal | 因果関係 | 明示的因果表現の検出 |
| comparison | 比較関係 | 比較対象としての言及 |
| semantic_manual | 意味的関連（手動） | 人間による関係付与 |

### D.3 エッジタイプ分布

| タイプ | 件数 | 割合 |
|--------|------|------|
| reference | 282 | 33.3% |
| subsumption | 247 | 29.2% |
| definition_reference | 162 | 19.1% |
| semantic_manual | 100 | 11.8% |
| comparison | 8 | 0.9% |
| その他 | 47 | 5.6% |

## F. 品質改善プロセス

### F.1 発見された問題と対応

初期抽出後、以下の品質問題が発見され、順次改善を行った。

| # | 問題 | 発見時期 | 対応スクリプト | 結果 |
|---|------|---------|---------------|------|
| 1 | 出典情報欠落（76%が「-」表示） | Phase 4後 | `15_restore_sources.py` | 473件修復 |
| 2 | 孤立ノード過多（40%） | Phase 4後 | `16_regenerate_gaps.py`, `17_add_relationships.py` | 16.4%に改善 |
| 3 | ギャップがブラックリスト経由のみ | Phase 5後 | `16_regenerate_gaps.py` | 多様化達成 |
| 4 | 重複概念の存在 | Phase 4後 | `11_detect_duplicates.py`, `14_apply_merge.py` | 24件マージ |
| 5 | JP/EU/US概念の孤立 | Phase 4後 | `17_add_relationships.py` | 39関係追加 |

### F.2 孤立率改善の経緯

知識グラフの孤立率（エッジを持たないノードの割合）は、分析の信頼性に直結する。以下の3段階で改善を行った。

| 段階 | 手法 | 孤立率 | エッジ数 | 平均次数 |
|------|------|--------|---------|---------|
| 初期状態 | 共起関係のみ | 40.0% | 584 | 1.86 |
| 改善1 | 定義文ベースのエッジ追加 | 26.5% | 746 | 2.38 |
| 改善2 | 手動関係追加（39件） | 16.4% | 846 | 2.72 |

**改善1: 定義文ベースのエッジ追加**

概念の定義文に他の概念名が含まれている場合、それらの間にエッジを追加した。

対象キー概念（18種）：
- 社会信用システム、連合懲戒、ブラックリスト、芝麻信用
- 信用修復、レッドリスト、失信被執行人、信用中国
- 統一社会信用コード、征信、人民銀行、NDRC、国務院
- 高消費制限、データフロー、ガバナンス、アルゴリズム、監視

**改善2: 手動関係追加**

JP/EU/US関連の孤立概念を中心に、人間の判断により意味的関連を追加した。

追加例：
- 日本信用情報機構（JICC） → 信用情報機関（上位概念）
- GDPR → データフロー規制（比較関係）
- 芝麻信用 → FICOスコア（比較関係）

### F.3 重複検出・マージプロセス

**手法**: Sentence-BERT（`paraphrase-multilingual-mpnet-base-v2`）を用いて、概念名のコサイン類似度を計算。閾値0.85以上のペアを重複候補として抽出。

**マージ例**:

| マージ元 | マージ先 | 類似度 | 理由 |
|----------|----------|--------|------|
| SCS-0071（信用中国-代替名） | SCS-0011（信用中国） | 0.92 | 同一概念の別名 |
| SCS-0200（信用中国-公的信用DB） | SCS-0011（信用中国） | 0.89 | 同一概念の別表現 |
| SCS-0091（連合懲罰） | SCS-0024（連合懲戒） | 0.95 | 表記揺れ |

**マージ統計**:
- Phase 1マージ: 11件
- Phase 2マージ: 8件
- Phase 3マージ: 5件
- 合計: 24件（627→622概念に統合）

### F.4 ギャップ検出アルゴリズムの改善

初版のギャップ検出では、ハブノード（ブラックリスト等）を経由するギャップが100%を占め、多様性に欠けていた。

**改善アルゴリズム（v2）**:
1. ハブノード除外: degree > 15のノードを橋渡し概念から除外
2. 多様性確保: 同一の橋渡し概念Bは最大5件まで
3. 異国間優先: country_tagが異なるA-C間のギャップを優先スコアリング

**改善結果**:
- 異国間ギャップ: 0% → 100%（50件中50件）
- 橋渡し概念の種類: 1種 → 23種

## H. 拡張可能性

本研究で構築した知識グラフは、以下の形で将来的な研究・実務に活用可能である。

### H.1 RAG（Retrieval-Augmented Generation）への応用

知識グラフをベクトルDBと組み合わせることで、SCS関連の質問応答システムを構築できる。

**想定アーキテクチャ**:
```
ユーザー質問
    ↓
[1] 質問をベクトル化
    ↓
[2] 知識グラフから関連概念を検索（グラフ探索 + ベクトル類似度）
    ↓
[3] 関連概念の定義・出典・接続関係をコンテキストとして取得
    ↓
[4] LLMが回答を生成（出典付き）
```

**利点**:
- 622概念の定義・関係性が構造化されており、ハルシネーション抑制に寄与
- 出典情報（88文献）が紐づいているため、回答の根拠を提示可能
- 4軸タグにより、ガバナンス・データフロー等の観点別検索が可能

### H.2 政策提案の関連概念可視化

新しい政策提案や制度設計案をノードとして知識グラフに追加することで、既存概念との関連性を可視化できる。

**活用例**:

| 新規ノード（提案） | 接続先（既存概念） | 関係タイプ | 示唆 |
|-------------------|-------------------|-----------|------|
| 日本版信用修復制度 | 信用修復（CN）、GDPR忘れられる権利（EU） | comparison | 中国・EUの先行事例を参照可能 |
| 法人番号活用拡大 | 統一社会信用コード（CN）、信用中国 | comparison | 中国の統合PFとの差異を確認 |
| 民間スコア規制 | 芝麻信用、FICOスコア、Yahoo!スコア撤退 | comparison | 失敗事例・成功事例の両方を参照 |

**操作手順**:
1. 新規概念をJSON形式で定義（concept_type, four_axis, definition等）
2. `17_add_relationships.py`で既存概念との関係を追加
3. `08_build_visualization.py`で可視化を更新
4. グラフ上で新規ノードの位置（近接概念）を確認

### H.3 継続的な文献追加

新たなSCS関連文献が公開された際、同じパイプラインで概念を追加できる。

```
新規PDF → 01_extract_text.sh → 02_chunk_texts.py → 概念抽出 → 03_merge_concepts.py
```

知識グラフは増分更新可能な設計となっており、研究の進展に応じて拡張できる。
