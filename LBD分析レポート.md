## 1. プロジェクト概要

### 1.1 目的

91件のPDF文献から概念を自動抽出し、知識グラフを構築することで、卒論の3つのリサーチクエスチョンに答えるエビデンスを体系化する。

## 2. 理論的背景

### 2.1 Swanson ABC モデル

本プロジェクトは Don R. Swanson（1986）が提唱した **Literature-Based Discovery (LBD)** を応用している。

**基本原理**:
```
A ─────────── C
 \           /
  \    ?    /
   \       /
    \     /
      B

A: 既知の概念群
B: 橋渡し概念（中間概念）
C: 別の既知概念群
?: 未検出の関係（研究仮説候補）
```

**ロジック**:
- AとCが直接接続されていない
- AとB、BとCが接続されている
- → AとCの間に未発見の関係がある可能性

### 2.2 元論文（Itani et al. 2025）との対応

本手法は超伝導材料発見論文（arXiv:2512.10847）のEnd-to-End LLMパイプラインを社会科学研究に応用したもの：

| 超伝導論文 | 本プロジェクト |
|-----------|---------------|
| 90,000 DOIs → 構造化DB | 91 PDFs → 概念DB |
| SC/NSC分類 | Task A: 企業影響分類 |
| Tc予測（回帰） | Task B: 4軸スコア回帰 |
| Tc→composition逆設計 | Task C: 制度移転可能性予測 |
| GNoMEスクリーニング | Swanson ABCギャップ検出 |

### 2.3 福田論文手法の適用

孤立概念の接続性向上のため、福田ら（2024）の知識グラフ自動構築手法を適用：

| 手法 | 内容 | 結果 |
|------|------|------|
| Sentence-BERT重複検出 | 埋め込みベクトルのcos類似度で同義語候補を検出 | 7件検出 |
| is-a関係推定 | 概念ペアの上位/下位関係をヒューリスティックで推定 | 8関係推定 |
| 手動関係追加 | JP/EU/US孤立概念の接続 | 39関係追加 |

**使用モデル**: `paraphrase-multilingual-mpnet-base-v2`（日中英対応）

## 3. パイプライン設計

### 3.1 全体フロー

```
[Phase 1] PDF → pdftotext → テキスト
    ↓ 目的: コンテキスト爆発回避
[Phase 2] テキスト → 3000字チャンク分割
    ↓ 目的: Claudeのコンテキスト上限内で処理
[Phase 3] チャンク → 概念抽出 + ラベル付与
    ↓ 目的: SCS関連の制度要素を構造化
[Phase 4] 概念DB統合 → 知識グラフ生成
    ↓ 目的: 概念間の関係を可視化・分析
[Phase 5] マルチタスク予測（Task A/B/C）
    ↓ 目的: RQ1-3に直接答えるエビデンス生成
[Phase 6] ギャップ検出（Swanson ABC）+ 仮説生成
    ↓ 目的: 未検出の関係から研究仮説を導出
[Phase 7] 可視化 + RQマッピング
    ↓ 目的: 卒論執筆に使える形で整理
```

### 3.2 コンテキスト管理の設計原則

**問題**: 91件のPDF（232MB, 510万字）を直接処理するとコンテキスト爆発

**解決策**:
1. **即時永続化**: 抽出結果は即座にファイル保存、コンテキストに蓄積しない
2. **バッチ分割**: 大きな処理は固定サイズバッチに分割
3. **ステートレス再開**: progress.jsonで状態管理、中断後も再開可能
4. **1文献1セッション**: 大規模文献（>100チャンク）は独立セッションで処理

### 3.3 概念抽出スキーマ

```json
{
  "concept_id": "SCS-0001",
  "jp_name": "連合懲戒",
  "en_name": "Joint Punishment",
  "zh_name": "联合惩戒",
  "concept_type": "mechanism | theory | phenomenon | actor | policy | data_system",
  "definition": "一箇所の信用失墜が複数領域に波及する制裁メカニズム",
  "four_axis": ["インセンティブ設計", "ガバナンス"],
  "source_papers": ["bi_2021", "liang_2018"],
  "context_quotes": ["..."],
  "related_concepts": ["ブラックリスト", "失信被執行人リスト"]
}
```

### 3.4 概念タイプ定義

| タイプ | 説明 | 例 |
|--------|------|-----|
| theory | 理論・概念枠組み | market credit, xingyong |
| mechanism | 制度メカニズム | 連合懲戒、信用修復 |
| policy | 政策・法規 | Planning Outline 2014-2020 |
| data_system | データシステム | 信用中国、芝麻信用 |
| actor | アクター | NDRC、人民銀行 |
| phenomenon | 現象・状況 | 三角債務、信用ギャップ |

### 3.5 エッジタイプ定義

| 関係タイプ | 説明 | 重み付け |
|-----------|------|---------|
| co_occurrence | 同一文献内共起 | 共起回数 |
| causal | 因果関係 | 明示的因果表現の有無 |
| comparison | 比較関係 | 比較対象として言及 |
| implementation | 実装関係 | 制度が他の制度を実装 |
| subsumption | 包含関係 | 上位概念-下位概念 |

## 11. SCS誤解の三層構造分析

### 11.1 発見の概要

手動統合作業中に、SCSに関する重要な発見があった。複数の一次資料を検証した結果、西洋メディアで描かれるSCS像と実態との間に大きな乖離が存在することが明らかになった。

**三層構造モデル**:

| 層 | 内容 | 特徴 |
|---|---|---|
| 第1層：西洋メディアの描写 | AIによるリアルタイム監視・自動制裁 | ディストピア的、技術決定論的 |
| 第2層：中国政府のレトリック | データ駆動型ガバナンスの理想像 | 野心的、プロパガンダ的 |
| 第3層：実装の実態 | ローテク・人的依存・断片化 | Excel、紙とペン、人的判断 |

### 11.2 主要な発見

**1. 自動制裁の不在**

> "there is no known instance in which automated data collection leads to the automated application of sanctions without the intervention of human regulators."
> — Trivium China (2020)

**2. ローテク実装**

> "The Social Credit System remains the least digitised of China's tech-driven monitoring and surveillance initiatives. It relies heavily on human investigations, reports, and decisions."
> — Drinhausen & Brussee, MERICS (2021)

**3. 栄成市の実態**

> 「栄成市では人間の『情報収集員』が町を歩き回り、紙とペンを使って人々の不正行為を書き留めていた」
> — MIT Technology Review (2023)

| ファイル | 内容 |
|----------|------|
| `scs_misconceptions_analysis.md` | 論文用テキスト（引用付き） |
| 知識グラフ | 誤解ハブノード構造 |

### 11.4 卒論への示唆

この発見はRQ1（4軸フレームワーク分析）に直接関係する：

1. **ガバナンス軸**: 実態は「データ駆動型」ではなく「人的依存型」
2. **データフロー軸**: 統合されたデータシステムは未完成、断片化が進行
3. **研究上の意義**: 西洋文献のSCS描写を鵜呑みにせず、一次資料に基づく分析が必要
